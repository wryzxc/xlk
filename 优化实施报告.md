#### 2.1.3 安全响应头增强

**问题描述**: 现有安全头仅包含基础的X-Content-Type-Options、X-Frame-Options、X-XSS-Protection，缺少Referrer-Policy和Permissions-Policy。

**实施方案**:
```python
@app.after_request
def add_security_headers(response):
    response.headers['X-Content-Type-Options'] = 'nosniff'
    response.headers['X-Frame-Options'] = 'SAMEORIGIN'
    response.headers['X-XSS-Protection'] = '1; mode=block'
    response.headers['Referrer-Policy'] = 'strict-origin-when-cross-origin'
    response.headers['Permissions-Policy'] = 'geolocation=(), microphone=(), camera=()'
    return response
```

**验证结果**: 所有响应头正确设置，通过安全扫描验证。

### 2.3 第三阶段：安全加固与后端改进

#### 2.3.1 代码执行安全沙箱升级（P0-高优先级）

**问题描述**: 原有代码执行仅使用简单的字符串关键字过滤（`'import os' in code`），可被多种方式绕过（字符串拼接、编码混淆等）。

**实施方案**:

1. **AST语法树分析**:
```python
import ast
import re

def validate_code_safety(code):
    """使用AST分析进行代码安全验证"""
    try:
        tree = ast.parse(code)
    except SyntaxError as e:
        return False, f'语法错误: {e}'
    
    dangerous_names = {
        'open', 'eval', 'exec', 'compile', '__import__',
        'input', 'os', 'sys', 'subprocess', 'socket',
        'urllib', 'requests', 'pickle', 'marshal',
        'ctypes', 'importlib', 'builtins'
    }
    
    for node in ast.walk(tree):
        # 检查导入语句
        if isinstance(node, ast.Import):
            for alias in node.names:
                if alias.name.split('.')[0] in dangerous_names:
                    return False, f'不允许导入模块: {alias.name}'
        
        # 检查危险函数调用
        if isinstance(node, ast.Call):
            if isinstance(node.func, ast.Name):
                if node.func.id in dangerous_names:
                    return False, f'不允许使用函数: {node.func.id}'
    
    # 正则检查字符串拼接绕过
    patterns = [
        r'__\s*import\s*__',
        r'eval\s*\(',
        r'exec\s*\(',
        r'open\s*\(',
        r'subprocess',
        r'os\s*\.\s*system',
    ]
    for pattern in patterns:
        if re.search(pattern, code, re.IGNORECASE):
            return False, '检测到可疑代码模式'
    
    return True, None
```

2. **执行端点增强**:
```python
@app.route('/run_code', methods=['POST'])
@login_required
def run_code():
    code = request.json.get('code', '')
    
    # 代码长度限制
    if len(code) > 5000:
        return jsonify({'success': False, 'error': '代码长度超过限制'})
    
    # AST安全分析（替代简单关键字过滤）
    is_safe, error_msg = validate_code_safety(code)
    if not is_safe:
        return jsonify({'success': False, 'error': f'代码安全检查失败: {error_msg}'})
    
    # ... 执行代码 ...
```

**安全防护能力**:

| 攻击方式 | 原方案 | 新方案 | 防护效果 |
|----------|--------|--------|----------|
| `import os` | 拦截 | 拦截 | 通过 |
| `eval("1+1")` | 拦截 | 拦截 | 通过 |
| `__im`+`port__("os")` | **绕过** | 拦截 | 通过 |
| `import o\ns` | **绕过** | 拦截 | 通过 |
| `getattr(__builtins__, 'ope'+'n')` | **绕过** | 拦截 | 通过 |

#### 2.3.2 会话安全配置

**问题描述**: 原有会话配置使用默认设置，缺少HttpOnly、SameSite、过期时间等安全属性。

**实施方案**:
```python
app.config.update(
    SESSION_COOKIE_SECURE=False,      # 开发环境关闭，生产环境设为True
    SESSION_COOKIE_HTTPONLY=True,     # 防止XSS窃取会话
    SESSION_COOKIE_SAMESITE='Lax',    # CSRF防护
    PERMANENT_SESSION_LIFETIME=timedelta(hours=2)  # 会话2小时过期
)
```

**安全收益**:
- **HttpOnly**: 防止JavaScript通过document.cookie窃取会话
- **SameSite=Lax**: 防止CSRF跨站请求伪造
- **2小时过期**: 限制会话被盗后的有效时间窗口

#### 2.3.3 密钥与环境变量管理

**问题描述**: SECRET_KEY和数据库URI硬编码在源代码中，存在泄露风险。

**实施方案**:
```python
# 优化前
app.config['SECRET_KEY'] = 'python-learning-platform-2024'
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///platform.db'

# 优化后
app.config['SECRET_KEY'] = os.environ.get('SECRET_KEY') or 'python-learning-platform-2024-dev'
app.config['SQLALCHEMY_DATABASE_URI'] = os.environ.get('DATABASE_URL') or 'sqlite:///platform.db'
```
