### 问题1: SQLAlchemy LegacyAPIWarning警告

**问题描述**: 
系统启动时出现 `LegacyAPIWarning: The Query.get() method is considered legacy as of the 1.x series of SQLAlchemy` 警告

**影响范围**: 
- 后端稳定性
- 未来SQLAlchemy 2.0升级兼容性
- 日志噪音

**根因分析**: 
使用了SQLAlchemy 1.x的 `query.get()` 方法，该方法在2.0版本中已标记为legacy

**解决方案**: 
- 将 `User.query.get(int(user_id))` 替换为 `db.session.get(User, int(user_id))`
- 将 `Exercise.query.get(sub.exercise_id)` 替换为 `db.session.get(Exercise, sub.exercise_id)`
- `.query.get_or_404()` 保持不变（Flask-SQLAlchemy封装，兼容2.0）

**修改文件**: 
- `app.py`: 第52行, 第346行

**验证结果**: 
✅ 服务器重启无警告，功能正常运行

**责任人**: AI Assistant
**完成时限**: 2026-04-28（已完成）

### 问题2: 移动端导航占用屏幕空间

**问题描述**: 
移动端导航菜单展开后采用向下展开方式，占用大量垂直空间，影响内容浏览

**影响范围**: 
- 移动端用户体验（768px以下屏幕）
- 导航易用性
- 内容可读性

**根因分析**: 
原有实现使用 `position:absolute; top:100%` 的下拉方式，在移动端展开后遮挡内容

**解决方案**: 
实现侧边抽屉式导航（Slide-out Drawer Menu）
- 固定定位左侧滑出面板（`position:fixed; left:0; top:0; bottom:0`）
- 宽度280px，最大85vw
- 添加遮罩层（overlay）点击关闭
- 滑动动画（`transform:translateX(-100%)` → `translateX(0)`）
- 支持ESC键关闭
- 关闭时恢复页面滚动

**修改文件**: 
- `base.html`: 添加遮罩层元素 `nav-overlay`
- `main.min.css`: 移动端导航样式重构（侧边抽屉+遮罩）
- `main.min.js`: 移动端菜单交互逻辑（open/close overlay）

**关键代码**:
```css
.nav-links {
  position: fixed;
  top: 0; left: 0; bottom: 0;
  width: 280px;
  transform: translateX(-100%);
  transition: transform .3s ease;
}
.nav-links.show { transform: translateX(0); }

.nav-overlay {
  position: fixed;
  inset: 0;
  background: rgba(0,0,0,.5);
  opacity: 0;
  pointer-events: none;
}
.nav-overlay.show {
  opacity: 1;
  pointer-events: auto;
}
```

**验证结果**: 
✅ 移动端导航流畅，不遮挡内容，支持键盘关闭

**责任人**: AI Assistant
**完成时限**: 2026-04-28（已完成）
