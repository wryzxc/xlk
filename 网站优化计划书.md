## 一、网站现状分析

### 1.1 技术架构审查

#### 1.1.1 后端架构

| 组件        | 现状                                | 评估                   |
| --------- | --------------------------------- | -------------------- |
| **Web框架** | Flask 3.0.0 (Python)              | 轻量灵活，适合中小型应用         |
| **ORM**   | Flask-SQLAlchemy 3.1.1            | 成熟稳定，但缺少连接池优化        |
| **认证**    | Flask-Login 0.6.3                 | 基础功能完善，缺少OAuth/SSO   |
| **数据库**   | SQLite (本地文件)                     | 开发便捷，但并发性能有限         |
| **表单验证**  | Flask-WTF + WTForms               | 基础功能满足需求             |
| **密码安全**  | Werkzeug generate\_password\_hash | PBKDF2算法，建议升级至Argon2 |

**架构优势**:

- 模块化设计，`course_data.py` 实现数据与逻辑分离
- 渐进式学习路径设计（lesson prerequisite chains）
- 完善的数据模型：User, Course, Lesson, Exercise, Exam, Progress 等

**架构缺陷**:

- SQLite 单文件数据库，不适合高并发生产环境
- 缺少缓存层（Redis/Memcached）
- 无API限流机制，存在暴力破解风险
- 代码执行使用 `subprocess.run`，沙箱隔离不足
- 无异步任务队列（Celery/RQ）

#### 1.1.2 前端架构

| 组件                 | 现状                         | 评估                 |
| ------------------ | -------------------------- | ------------------ |
| **CSS框架**          | 自定义CSS (main.min.css)      | 设计系统完整，但无Utility类库 |
| **JS框架**           | 原生JavaScript (main.min.js) | 轻量但缺少现代框架特性        |
| **模板引擎**           | Jinja2                     | 服务端渲染，SEO友好        |
| **Service Worker** | 基础缓存策略                     | 已实现但缓存范围有限         |
| **字体加载**           | Google Fonts (preconnect)  | 有优化但仍有阻塞风险         |

**前端优势**:

- CSS设计系统完善（色彩/阴影/圆角/间距体系化）
- WCAG 2.1 AA 标准色彩对比度
- 响应式布局支持
- PWA基础（Service Worker + manifest）

**前端缺陷**:

- 无代码分割（Code Splitting）
- 缺少懒加载（Lazy Loading）实现
- 无骨架屏（Skeleton Screen）
- JavaScript未压缩混淆（main.min.js 实际未真正minify）
- 缺少前端路由缓存策略

#### 1.2.1 当前性能瓶颈

| 指标项                | 现状     | 目标值   | 差距    |

| **首屏加载时间 (FCP)**   | \~2.5s | <1.5s | -1.0s |

#### 1.2.2 性能问题根因分析

1. **渲染阻塞资源**
   - Google Fonts 同步加载阻塞首屏
   - CSS文件未实现Critical CSS内联
   - JavaScript无 `defer`/`async` 属性
2. **资源加载优化不足**
   - 图片无 WebP/AVIF 格式支持
   - 缺少资源预加载（Preload/Prefetch）
   - Service Worker 缓存策略过于简单
3. **服务端性能**
   - SQLite 无连接池，高并发下性能急剧下降
   - 无查询缓存，重复查询开销大
   - 代码执行使用子进程，资源消耗高
4. **网络传输**
   - 无Gzip/Brotli压缩配置
   - 静态资源未使用CDN
   - HTTP/2 未启用

**策略8: 缓存策略优化**

```python
# 当前缓存配置
app.config['SEND_FILE_MAX_AGE_DEFAULT'] = 31536000

# 优化: 分层缓存策略
@app.after_request
def optimize_caching(response):
    if request.path.startswith('/static/'):
        if '.min.' in request.path or request.path.endswith(('.css', '.js')):
            # 版本化资源长期缓存
            response.headers['Cache-Control'] = 'public, max-age=31536000, immutable'
            response.headers['Vary'] = 'Accept-Encoding'
        elif request.path.endswith(('.png', '.jpg', '.webp', '.avif')):
            # 图片中期缓存
            response.headers['Cache-Control'] = 'public, max-age=2592000'
    return response
```

**策略16: 动态Sitemap生成**

```python
from flask import make_response
from xml.etree.ElementTree import Element, SubElement, tostring

@app.route('/sitemap.xml')
def sitemap():
    urlset = Element('urlset', xmlns='http://www.sitemaps.org/schemas/sitemap/0.9')
    
    # 首页
    add_url(urlset, '/', priority='1.0', changefreq='daily')
    
    # 课程页
    for course in Course.query.filter_by(is_published=True).all():
        add_url(urlset, f'/course/{course.id}', priority='0.8', changefreq='weekly')
        for lesson in course.lessons:
            add_url(urlset, f'/lesson/{lesson.id}', priority='0.6', changefreq='monthly')
    
    # 练习页
    for exercise in Exercise.query.all():
        add_url(urlset, f'/exercise/{exercise.id}', priority='0.5')
    
    xml = tostring(urlset, encoding='unicode')
    response = make_response(xml)
    response.headers['Content-Type'] = 'application/xml'
    return response
```

### 3.5 安全加固

#### 3.5.1 代码执行沙箱

**策略20: 安全代码执行环境**

```python
import docker
from docker.errors import ImageNotFound

def run_code_sandbox(code: str, timeout: int = 5) -> dict:
    """
    使用Docker容器隔离执行用户代码
    替代当前的subprocess.run方式
    """
    client = docker.from_env()
    
    # 创建临时文件
    with tempfile.NamedTemporaryFile(mode='w', suffix='.py', delete=False) as f:
        f.write(code)
        temp_file = f.name
    
    try:
        container = client.containers.run(
            'python:3.10-slim',
            f'python /code/user_script.py',
            volumes={temp_file: {'bind': '/code/user_script.py', 'mode': 'ro'}},
            network_mode='none',  # 禁止网络访问
            mem_limit='64m',       # 内存限制
            cpu_quota=50000,       # CPU限制
            pids_limit=10,         # 进程数限制
            detach=True,
            auto_remove=True
        )
        
        result = container.wait(timeout=timeout)
        logs = container.logs().decode('utf-8')
        
        return {
            'success': result['StatusCode'] == 0,
            'output': logs,
            'return_code': result['StatusCode']
        }
    except Exception as e:
        return {'success': False, 'error': str(e)}
    finally:
        os.unlink(temp_file)
```

from argon2 import PasswordHasher
ph = PasswordHasher(time_cost=2, memory_cost=65536, parallelism=1)

def set_password(self, password):
    self.password_hash = ph.hash(password)

def check_password(self, password):
    try:
        ph.verify(self.password_hash, password)
        return True
    except:
        return False
