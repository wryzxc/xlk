## 一、项目技术栈分析

| 组件 | 技术 | 版本要求 |
|------|------|----------|
| 后端框架 | Flask | >= 2.0 |
| 数据库 | SQLite | 内置 |
| ORM | SQLAlchemy | >= 2.0 |
| 认证 | Flask-Login | >= 0.6 |
| 前端 | HTML + CSS + JavaScript | 现代浏览器 |

**部署特性：**
- 使用 SQLite 文件数据库，无需独立数据库服务
- 静态文件（CSS/JS/图片）由 Flask 提供
- 会话存储在客户端 Cookie 中
- 文件上传功能（可选配置云存储）

```bash
# 1. 确保所有依赖已记录

# Environment

# IDE
.vscode/
.idea/
*.swp
*.swo

# OS

Thumbs.db

# Uploads
uploads/
EOF

# 3. 检查环境变量配置
# 生产环境必须使用环境变量设置 SECRET_KEY
```

### 3.2 生产环境配置调整

修改 `app.py` 中的配置：

```python
import os

# 生产环境配置
app.config['SECRET_KEY'] = os.environ.get('SECRET_KEY')
app.config['SQLALCHEMY_DATABASE_URI'] = os.environ.get('DATABASE_URL', 'sqlite:///platform.db')
app.config['DEBUG'] = False

# 安全会话配置（生产环境）
app.config.update(
    SESSION_COOKIE_SECURE=True,       # 仅HTTPS传输
    SESSION_COOKIE_HTTPONLY=True,     # 防止XSS
    SESSION_COOKIE_SAMESITE='Lax',    # CSRF防护
    PERMANENT_SESSION_LIFETIME=timedelta(hours=2)
)
```

### 3.3 数据库初始化脚本

创建 `init_production_db.py`：

```python
import os
import sys

# 设置生产环境
os.environ['FLASK_ENV'] = 'production'

from app import app, db
from course_data import COURSES, LESSONS, EXERCISES, EXAMS, EXAM_QUESTIONS

def init_db():
    with app.app_context():
        # 创建所有表
        db.create_all()
        
        # 检查是否已有数据
        from app import User
        if User.query.first() is None:
            print("初始化数据...")
            # 导入初始化逻辑
            from init_db import init_database
            init_database()
            print("数据初始化完成")
        else:
            print("数据库已有数据，跳过初始化")

if __name__ == '__main__':
    init_db()
```

### 3.4 入口文件配置

创建 `wsgi.py`（WSGI 入口）：

```python
import os
from app import app

if __name__ == "__main__":
    app.run()
```

创建 `Procfile`（Heroku/Render 需要）：

```
web: gunicorn wsgi:app
```

## 四、平台部署方案

### 4.1 Render（推荐 - 完全免费）

**特点：**
- 免费层级：Web服务永久运行
- 支持自定义域名
- 自动 HTTPS
- Git 自动部署

**部署步骤：**

1. 注册账号：https://render.com（使用 GitHub 账号登录）

2. 创建 New Web Service

3. 连接 GitHub 仓库

4. 配置构建参数：

```yaml
# render.yaml（可选，用于基础设施即代码）
services:
  - type: web
    name: python-learning-platform
    env: python
    buildCommand: "pip install -r requirements.txt"
    startCommand: "gunicorn wsgi:app"
    envVars:
      - key: PYTHON_VERSION
        value: 3.10.0
      - key: SECRET_KEY
        generateValue: true
      - key: FLASK_ENV
        value: production
```

5. 添加环境变量：
   - `SECRET_KEY`：随机生成的强密钥
   - `DATABASE_URL`：SQLite 使用默认路径

6. 点击 Deploy，等待自动构建完成

**访问地址：** `https://your-service-name.onrender.com`

**特点：**
- 免费账户：1个Web应用
- 提供 Bash 控制台
- 内置 Python 环境

**部署步骤：**

### 4.4 Vercel（适合静态网站 + Serverless）

**特点：**
- 全球 CDN 加速
- 自动 HTTPS
- 适合前后端分离架构

**部署步骤（使用 Flask + Vercel Serverless）：**

1. 安装 Vercel CLI：

```bash
npm i -g vercel
```

2. 创建 `vercel.json`：

```json
{
  "version": 2,
  "builds": [
    {
      "src": "api/index.py",
      "use": "@vercel/python"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "api/index.py"
    }
  ]
}
```

3. 创建 `api/index.py`：

```python
from app import app

# Vercel 需要这个 handler
from http.server import BaseHTTPRequestHandler

class handler(BaseHTTPRequestHandler):
    def do_GET(self):
        return app(self.environ, self.start_response)
```

4. 部署：

```bash
vercel --prod
```

> 注意：Vercel 的免费层级有无服务器函数执行时间限制（10秒），不适合长时间运行的代码执行功能。

### 4.6 GitHub Pages（纯静态网站）

**适用场景：**
- 纯 HTML/CSS/JS 项目
- 文档网站
- 个人博客

**部署步骤：**

1. 启用 GitHub Pages：
   - 进入仓库 Settings -> Pages
   - Source 选择 Deploy from a branch
   - 选择 main 分支和 /root 目录

2. 或使用 GitHub Actions 自动部署：

```yaml
# .github/workflows/pages.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      
      - name: Build static site
        run: |
          pip install flask frozen-flask
          python build_static.py
      
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./build
```

> 注意：GitHub Pages 仅支持静态内容，Flask 应用需要预渲染为静态 HTML。

## 五、常见问题排查

### 5.1 应用无法启动

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| `ModuleNotFoundError` | 依赖未安装 | 运行 `pip install -r requirements.txt` |
| `Address already in use` | 端口被占用 | 更换端口或杀死占用进程 |
| `SECRET_KEY not set` | 环境变量缺失 | 设置 `SECRET_KEY` 环境变量 |
| `Permission denied` | 文件权限问题 | `chmod 755` 设置正确权限 |

### 5.2 数据库问题

```bash
# 数据库锁定（SQLite并发问题）
# 解决方案：使用连接池或减少并发

# 数据库迁移
flask db init      # 初始化迁移
flask db migrate   # 创建迁移脚本
flask db upgrade   # 应用迁移

# 数据库备份
cp platform.db platform.db.backup.$(date +%Y%m%d)
```

### 5.3 静态文件 404

```python
# 确保配置正确
app = Flask(__name__, static_folder='static', static_url_path='/static')

# 生产环境使用 Nginx 提供静态文件
# 配置见 4.7 节 Nginx 配置
```

### 5.4 内存不足

```bash
# 减少 Gunicorn worker 数量
# 公式：workers = 2 * CPU核心数 + 1
gunicorn -w 2 -b 0.0.0.0:8000 wsgi:app

# 或限制每个 worker 的内存
ulimit -v 512000  # 限制 512MB
```

## 七、安全配置

### 7.1 必做安全设置

```python
# 1. 强密钥（生产环境）
import secrets
SECRET_KEY = secrets.token_hex(32)

# 2. 安全头部
@app.after_request
def add_security_headers(response):
    response.headers['X-Content-Type-Options'] = 'nosniff'
    response.headers['X-Frame-Options'] = 'SAMEORIGIN'
    response.headers['X-XSS-Protection'] = '1; mode=block'
    response.headers['Referrer-Policy'] = 'strict-origin-when-cross-origin'
    response.headers['Permissions-Policy'] = 'geolocation=(), microphone=(), camera=()'
    response.headers['Content-Security-Policy'] = "default-src 'self'; script-src 'self' 'unsafe-inline'"
    return response

# 3. 密码安全
# 使用 Werkzeug 的 generate_password_hash（已自动使用 PBKDF2）

# 4. 输入验证
from werkzeug.utils import secure_filename
import re

def validate_username(username):
    return re.match(r'^[a-zA-Z0-9_]{3,20}$', username) is not None

def safe_filename(filename):
    return secure_filename(filename)
```

### 7.2 代码执行沙箱

```python
import ast
import sys

# 限制可访问的内置函数
SAFE_BUILTINS = {
    'abs', 'all', 'any', 'bin', 'bool', 'chr', 'dict', 'dir',
    'divmod', 'enumerate', 'filter', 'float', 'format', 'frozenset',
    'hasattr', 'hash', 'hex', 'id', 'input', 'int', 'isinstance',
    'issubclass', 'iter', 'len', 'list', 'map', 'max', 'min',
    'next', 'oct', 'open', 'ord', 'pow', 'print', 'range',
    'repr', 'reversed', 'round', 'set', 'slice', 'sorted',
    'str', 'sum', 'tuple', 'type', 'vars', 'zip'
}

def is_safe_code(code):
    """检查代码是否包含危险操作"""
    try:
        tree = ast.parse(code)
    except SyntaxError:
        return False
    
    for node in ast.walk(tree):
        if isinstance(node, (ast.Import, ast.ImportFrom)):
            return False
        if isinstance(node, ast.Call):
            if isinstance(node.func, ast.Name):
                if node.func.id not in SAFE_BUILTINS:
                    return False
    return True
```

## 八、监控与维护

### 8.1 日志监控

```python
import logging
from logging.handlers import RotatingFileHandler

# 配置日志
if not app.debug:
    file_handler = RotatingFileHandler(
        'logs/platform.log',
        maxBytes=10240000,  # 10MB
        backupCount=10
    )
    file_handler.setLevel(logging.INFO)
    file_handler.setFormatter(logging.Formatter(
        '%(asctime)s %(levelname)s: %(message)s [in %(pathname)s:%(lineno)d]'
    ))
    app.logger.addHandler(file_handler)
    app.logger.setLevel(logging.INFO)
    app.logger.info('Platform startup')
```
